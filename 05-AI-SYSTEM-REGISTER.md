# 05 — AI System Register

**EU AI Act Reference:** Articles 49, 71 | Article 16(d) (Provider obligation to keep register)  
**Applies to:** All organisations providing or deploying AI systems in the EU  
**Last Updated:** April 2026

---

## Purpose

This register serves as the organisation's central inventory of all AI systems. It supports:

- EU AI Act risk classification and compliance tracking
- Provider obligations to maintain a register (Article 16(d))
- Deployer obligations to record systems in use (Article 26(6))
- Internal governance and audit readiness
- Preparation for mandatory EU database registration (Article 71)

---

## Instructions

1. Register **all** AI systems — regardless of risk tier
2. Complete a [Risk Classification (01)](01-RISK-CLASSIFICATION-GUIDE.md) for each system
3. Update the register whenever a system is: acquired, modified, decommissioned, or reclassified
4. Review the complete register at minimum **annually** or upon regulatory changes
5. For High-Risk systems: verify EU database registration number is recorded

## Field Definitions

| Field | Definition |
|-------|-----------|
| **System ID** | Unique internal identifier (format: AI-EU-NNN) |
| **System Name** | Descriptive name of the AI system |
| **Version** | Current production version |
| **Provider / Vendor** | External provider or "Internal" if built in-house |
| **Owner (Internal)** | Business owner / accountable team |
| **Business Function** | e.g. HR, Finance, Operations, Customer Service |
| **Primary Use Case** | Brief description of what the system does |
| **Risk Tier** | Unacceptable / High / Limited / Minimal |
| **Annex III Category** | Area 1–8 if High Risk (e.g. "Area 4 — Employment") |
| **EU DB Registration #** | Number from EU AI database (mandatory for High Risk) |
| **FRIA Required** | Yes / No (required for certain public sector deployers) |
| **FRIA Status** | Not started / In progress / Completed / Review due |
| **Conformity Assessment** | Not started / In progress / Completed / Certified |
| **Technical Docs Complete** | Yes / No / Partial |
| **Deployment Date** | Date system went live in production |
| **Review Date** | Next scheduled review / reassessment date |
| **Status** | Active / Decommissioned / Paused / In Development |
